# 🧠 智能技術教學框架
**基於認知科學的決策樹驅動教學方法論**

> 從電子書的知識分類理論，到革命性的技術教學實踐工具

## 📖 專案起源

這個專案始於對一本認知心理學電子書截圖的深度分析。書中討論了**四種知識類型**（Factual, Conceptual, Procedural, Metacognitive）和**資訊處理模型**。我們將這些學術理論轉化為實用的技術教學方法論，最終發展出一套完整的智能教學生態系統。

## 🎯 核心理念

### 傳統技術教學的痛點
- ❌ **知識灌輸**：大量概念轟炸，學習者消化不良
- ❌ **記憶導向**：強調背誦指令，忽視理解和應用
- ❌ **缺乏實戰**：理論與實際工作場景脫節
- ❌ **一刀切**：忽視學習者個體差異和學習狀態

### 我們的解決方案
- ✅ **Procedural 先行**：先體驗操作，再理解概念
- ✅ **漸進式概念導入**：在需要時才引入，避免認知超載
- ✅ **決策樹驅動**：培養判斷能力，而非僅僅記憶
- ✅ **智能引導**：動態適應學習者狀態，個人化教學

## 🚀 核心特色

### 🌳 決策樹驅動教學法
```
傳統方法：知識點A → 知識點B → 知識點C → 希望學生會應用
我們的方法：情境問題 → 決策流程 → 操作執行 → 概念理解 → 內化判斷
```

### 🤖 智能引導師系統
- **狀態識別**：即時判斷學習者當前狀態（順利/需要支援/困難/超前）
- **動態調節**：根據狀態調整教學節奏和策略
- **個人化引導**：針對不同學習風格調整溝通方式
- **情緒支援**：提供心理安全感和學習動機維持

### 🛡️ 學習透明度設計
- **完整 Cheatsheet**：一開始就提供所有學習內容，消除焦慮
- **學習地圖**：清晰的進度指示和階段目標
- **邊界設定**：明確告知不教的內容，避免期待落差

## 🛠️ 使用方式

### Quick Start

1. **選擇你要教學的技術主題**
   ```
   例如：Git 版本控制、React Hooks、Python 函數等
   ```

2. **使用 prompt-v4.md 設計教學內容**
   ```markdown
   將 {技術主題} 替換為你的主題
   按照框架分析四種知識類型
   設計對應的決策樹
   ```

3. **啟動智能引導師**
   ```markdown
   將設計好的教學內容提供給 intelligent-tutor.md
   設定學習者背景和目標
   開始動態互動式教學
   ```

### 實際應用範例

#### 範例 1：Git 衝突解決教學
```
🎯 目標：學會處理 Git merge conflict
⏱️ 時間：2小時 mini workshop
🌳 核心決策樹：「遇到 conflict 怎麼處理？」

Phase 1: 故意製造 conflict，讓學習者體驗
Phase 2: 建構 conflict 解決的決策流程  
Phase 3: 練習使用決策樹解決新 conflict
Phase 4: 發展預防和診斷能力
```

#### 範例 2：JavaScript 除錯技能
```  
🎯 目標：建立系統性的除錯能力
⏱️ 時間：1.5小時
🌳 核心決策樹：「程式不運作時如何診斷？」

從錯誤訊息 → 診斷原因 → 選擇解決策略 → 執行修復
```
